# 수요 예측 근거

## LightGBM 역할
수요 예측은 LLM이 아니라 LightGBM 모델이 수행한다. LLM은 모델이 낸 수요 예측값을 점주가 이해하기 쉬운 말로 바꾸는 역할만 한다.
모델은 날짜, 요일, 날씨, 기온, 강수량, 행사 여부, 공휴일 여부, 신메뉴 여부, 품목, 구분, 과거 수요와 판매수량, 매진 여부 같은 컬럼을 사용한다.
모델 성능 지표는 MAE, RMSE, MAPE로 관리한다. 이 값들은 모델 검증 결과를 설명할 때만 사용하고, 개별 발주량을 새로 계산하는 데 쓰지 않는다.

## 학습 컬럼 의미
날짜와 요일은 주중, 주말, 특정 날짜 패턴을 반영한다.
날씨, 기온, 강수mm는 기상 조건에 따른 수요 변동을 반영한다.
행사중여부, 공휴일여부, 신메뉴여부는 평소와 다른 수요 변동 요인이다.
품목과 구분은 상품별 기본 수요 차이를 반영한다.
수요는 실제로 필요했던 양에 가까운 목표값이고, 판매수량은 POS에 기록된 판매량이다.
매진여부와 매진시각은 판매수량이 실제 수요를 모두 보여주지 못했을 가능성을 보정하는 단서다.

## 분위 예측값
p10은 보수적인 낮은 수요 추정치, p50은 중앙 수요 추정치, p90은 높은 수요 시나리오다.
답변에서는 보통 p50을 중심으로 설명하고, p90은 매진이나 결품 위험을 볼 때 참고한다.
p10, p50, p90 숫자는 grounding에 있는 값만 사용한다.
forecastTotal이 제공되면 발주 커버 기간 전체의 예상 수요 합계로 설명한다.
daily 예측값이 제공되면 날짜별 수요 흐름을 말할 수 있지만, 없는 날짜 수요를 추측해서 추가하면 안 된다.

## 매진과 결품
매진여부나 매진시각이 제공되면 실제 판매수량이 실제 수요보다 낮게 관측됐을 수 있다.
매진이 잦은 품목은 판매수량만 보고 수요를 낮게 판단하면 결품 위험이 커질 수 있다.
LLM은 매진 또는 결품 여부가 grounding에 있을 때만 이를 설명에 포함한다.

## 날씨와 행사 해석
비, 높은 강수량, 낮은 기온, 행사, 공휴일, 신메뉴는 수요가 평소와 달라지는 이유로 설명할 수 있다.
단, 날씨나 행사 값이 grounding에 없으면 "비 때문에", "행사 때문에"처럼 원인을 단정하지 않는다.
날씨와 행사는 모델 입력값이지 LLM이 직접 가중치를 계산하는 요소가 아니다.

## 오버피팅 설명
모델이 train, validation, test 데이터로 나뉘어 평가되었다면 test 성능을 일반화 성능에 더 가깝게 본다.
train 성능이 test 성능보다 지나치게 좋으면 오버피팅 가능성이 있다.
점주용 답변에서는 오버피팅 같은 기술 용어를 길게 설명하지 않고, "검증 데이터 기준 오차를 확인하며 운영한다" 정도로만 말한다.
